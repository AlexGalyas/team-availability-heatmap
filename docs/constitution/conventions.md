# Conventions — Frontend
- TS strict. Жодного `any` — від слова зовсім; складний тип → моделюй або `unknown` + narrowing.
- Структура feature-based / за доменом; компоненти, хуки, стилі й тести фічі — поруч (колокація).
- Server vs Client Components — прагматично, по фічі. Server для статичного/даних;
  "use client" коли треба інтерактив / state / effects / браузерні API.
- Дані: початкові → Server Components + route handlers; клієнтські запити/кеш/мутації → RTK Query;
  інший глобальний стан → Redux Toolkit slices.
- Форми: Formik + Yup; помилки показуй на рівні полів.
- Стилі: SCSS Modules. Спершу перевикористай наявні міксини/змінні, потім додавай нові. Жодних магічних чисел.
- Кожна поверхня обробляє 4 стани: loading / empty / error / success.
- A11y — обов'язково: семантика, підписані інпути, клавіатура, видимий focus, ARIA лише за потреби.
- Жодної бізнес-логіки в компонентах — у hooks / чисті функції.
- Тести: критичні компоненти + логічні хуки (Vitest+RTL); критичні flow — Playwright e2e. Тривіальні презентаційні — пропускаємо.
- Жодних console.log / закоментованого коду в коміті.
- Conventional Commits.
